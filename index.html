6. Root Cause Analysis

6.1 Primary Causes Identified

1. Data Drift in Key Features

  • Transaction Type Distribution: CASH-OUT transactions increased by 34%, shifting risk profile
  • Amount Patterns: Average transaction amount increased 9.7%, affecting decision boundaries
  • User Demographics: Newer users (lower tenure) increased by 27.8%
  • Impact: Model trained on different population distribution, reducing predictive accuracy

2. Concept Drift

  • Fraud Tactics Evolution: Criminals adapted to existing model patterns
  • New Fraud Types: Emergence of micro-transaction fraud not in training data
  • Behavioral Shifts: Legitimate user behavior changed, creating new false positive patterns
  • Impact: Static model cannot adapt to evolving fraud landscape

3. Feature Distribution Shifts

  • Statistical Significance: KS-test p-values < 0.001 for top 3 features
  • Decision Boundary Impact: Optimal threshold shifted from 0.5 to 0.35
  • Feature Interactions: Correlation patterns between features changed significantly
  • Impact: Model's learned relationships no longer represent current data
